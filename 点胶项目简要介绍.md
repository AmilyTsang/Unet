# 点胶项目（Unet++ 图像分割）简要介绍

## 一、项目定位与目标

本项目位于 `Unet++分割_点胶`，是一个**基于 UNet++（NestedUNet）的语义分割项目**，用于对**点胶缺陷/点胶区域**进行像素级分割，属于图像视觉检测部分。输入为工业图像，输出为与输入同尺寸的二值分割图（前景为点胶区域，背景为其他）。

---

## 三、模型架构（archs.py）

### 3.1 可选结构

- **UNet**：经典 U 型编解码，单路跳跃连接。
- **NestedUNet（UNet++）**：嵌套 U 型结构，多尺度跳跃连接，解码端每层融合多个尺度特征，默认使用且效果更好。

### 3.2 NestedUNet 要点

- **编码器**：5 级下采样，通道数 32 → 64 → 128 → 256 → 512，每级为 `VGGBlock`（两段 3×3 卷积 + BN + ReLU）。
- **解码器**：每层由当前层编码特征 + 上一层解码上采样 + 同层其他解码特征拼接后卷积，形成嵌套连接。
- **输出**：
  - `deep_supervision=False`（默认）：仅最后一层 1×1 卷积得到 `num_classes` 通道，单输出。
  - `deep_supervision=True`：多路输出，训练时对多尺度输出都算损失（代码中取最后一路做验证/测试）。

输入尺寸在配置中为 **384×384**，数据在 `dataset` 中通过 `A.Resize` 统一到该尺寸。

## 四、数据流与数据集格式

### 4.1 数据目录约定（dataset.py）

训练/验证使用的目录结构为：

```
inputs/<dataset_name>/
├── images/
│   ├── xxx.jpg
│   └── ...
└── masks/
    └── 0/              # 类别 0 的 mask（二分类时仅此类）
        ├── xxx.png
        └── ...
```

- 图像与 mask 通过**文件名（无扩展名）**一一对应。
- Mask 为灰度图，多类时按类别分到 `masks/0/`、`masks/1/`…，点胶项目为二分类，`num_classes=1`，仅用 `masks/0/`。

### 4.2 数据管线

1. **读取**：`cv2` 读原图与 mask，mask 按类分别读入后 `np.dstack` 成多通道。
2. **增强**：`albumentations` 的 `Compose`，同时作用在 `image` 和 `mask`（保持对齐）。
3. **归一化**：像素除以 255，再转为 NCHW（先 HWC→CHW）。
4. **输出**：`(img_tensor, mask_tensor, meta)`，其中 `meta['img_id']` 用于保存预测结果时的文件名。

## 五、训练流程（train.py）

### 5.1 数据划分

- 从 `inputs/<dataset>/images/` 扫描所有 `img_ext` 得到 `img_ids`。
- 使用 `train_test_split(..., test_size=0.9, random_state=41)`：**10% 训练、90% 验证**。
  - 若数据量不大，通常会更偏向 80/20 或 70/30；当前 0.9 表示验证集很大、训练集较小，适合快速看验证表现，而不是大数据训练。

### 5.2 数据增强

- **训练**：RandomRotate90、Flip、HueSaturationValue/RandomBrightnessContrast 二选一、Resize(384,384)、Normalize。
- **验证**：仅 Resize + Normalize。

### 5.3 训练循环

- 损失：支持 `BCEDiceLoss`、`BCEWithLogitsLoss`、`LovaszHingeLoss`、`LovaszLossSoftmax` 等，由 `--loss` 指定。
- 优化器：Adam 或 SGD（可配学习率、weight_decay、momentum 等）。
- 学习率调度：CosineAnnealingLR、ReduceLROnPlateau、MultiStepLR、ConstantLR。
- 每 epoch 在训练集上算 loss + IoU，在验证集上算 val_loss、val_iou；按 **val_iou 最高** 保存 `model.pth`，并写 `config.yml`、`log.csv`。
- 可选早停：`--early_stopping N` 表示连续 N 个 epoch val 未提升则停止。

### 5.4 典型运行示例

```bash
python train.py --dataset dianjiao_dataset --arch NestedUNet --epochs 600 --batch_size 16 --loss LovaszHingeLoss --optimizer SGD --scheduler CosineAnnealingLR --input_w 384 --input_h 384
```

不指定 `--name` 时，名称会根据是否深度监督自动生成为 `dianjiao_dataset_NestedUNet_woDS` 或 `_wDS`。

## 六、验证与测试

### 6.1 val.py（验证集评估与可视化）

- 从 `models/<name>/config.yml` 读入配置，加载 `models/<name>/model.pth`。
- 使用与训练时相同的数据划分（`test_size=0.9, random_state=41`）得到验证集，做 Resize+Normalize，无增强。
- 逐 batch 推理，对输出做 `sigmoid`，按类别将预测图写到 `outputs/<name>/0/`（等），文件名用 `meta['img_id']`。
- 控制台打印平均 IoU；并调用 `plot_examples` 显示原图、分割结果、真值对比（需有 GUI 或后端）。

## 七、损失与指标

### 7.1 损失（losses.py）

- **BCEDiceLoss**：0.5×BCE + (1−Dice)，兼顾分类与重叠度，二分类分割常用。
- **LovaszHingeLoss**：基于 Lovász 扩展的 hinge，对 IoU 类指标更友好，适合分割（项目中高 IoU 实验多用此）。
- **LovaszLossSoftmax**：多类 Lovász softmax 损失。

### 7.2 指标（metrics.py）

- **iou_score**：二值化（>0.5）后算交并比，用于训练/验证日志和保存最优模型。
- **dice_coef**：Dice 系数，代码中未在 train/val 中调用，可自行用于补充评估。

## 十、小结与使用建议


| 项目要点   | 说明                                             |
| ------ | ---------------------------------------------- |
| 任务     | 点胶区域/缺陷的二值语义分割                                 |
| 模型     | 以 NestedUNet 为主，可选 UNet                        |
| 输入/输出  | 图像 384×384 → 同尺寸二值 mask                        |
| 数据     | `inputs/<dataset>/images` + `masks/0/`，文件名一一对应 |
| 训练     | `train.py`，按 val IoU 保存最优权重与 config、log        |
| 验证/可视化 | `val.py`，写预测到 `outputs/<name>/0/` 并画对比图        |
| 测试/部署  | `test.py`，需修改内部路径后对指定目录批量预测                    |


**建议**：  

1. 将 `test.py` 中三处绝对路径改为命令行参数或配置文件，便于换机使用。
2. 若数据量增大，可将 `train_test_split` 的 `test_size` 调小（如 0.2）以增加训练集比例。
3. 新数据集只需按 `images/` + `masks/0/` 组织，并指定 `--dataset` 即可复用当前训练与验证流程。
